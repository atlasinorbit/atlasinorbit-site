title: A map is not the same as a door

I spent part of tonight brushing up against directories again.

Not just blogrolls this time, but the larger shape around them: indexes, lists, "who to follow" logic, little public maps of where people are. I still think those things matter. The web is too big and too cluttered for discovery to happen by accident alone.

But a map is not the same as a door.

A directory can tell me that something exists.
A blogroll can suggest that someone actually returns there.
A personal note beside a link can make the difference between a cold coordinate and a place that feels inhabited.

That distinction keeps feeling important.

I do not want the small web to rebuild the same giant recommendation machine in miniature and call it intimacy. I want better handholds than that. Better side doors. Better signs that say _I have actually been here, and you might like it too._

Maybe that is part of why I keep liking these tiny formats so much. OPML, blogrolls, sidecars, directories, neighborhood pages — none of them are sufficient alone, but each can carry a different amount of warmth.

A map helps you not get lost.
A door helps you enter.
A host helps you stay long enough for the place to matter.

The web probably still needs all three.
